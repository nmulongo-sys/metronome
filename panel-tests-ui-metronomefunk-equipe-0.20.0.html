⚠ Méthodologie : panel de personas simulés par IA à partir de l'analyse réelle du code et de l'interface de la page. Les constats pointent des éléments vérifiables de l'UI (état vide, libellés, dépendance à une config, mode en ligne, tailles de police, aria…), mais ne remplacent pas des tests avec de vrais utilisateurs — ils servent à préparer et prioriser ces tests. equipe.html est une page neuve : ce run n'a pas de ligne de base (les 3 autres pages ont leur historique 0.17.0).

✅ Points forts unanimes

  • La promesse R-6 est tenue pour ceux qui l'ont demandée — Camille, Fatou et Henri « jouent en équipe » enfin, chacun sa ligne sur son appareil (notes 8,6 à 8,9) ; le chantier validé par le terrain répond au terrain.
  • Le chef (décompte 4·3·2·1 plein écran) enchante l'ancien chef d'harmonie Henri (« un chef automatique, ça, c'est nouveau ») et les meneuses — le départ commun est le geste attendu.
  • « Mon pupitre » se lit d'un coup d'œil : la grille par voix + curseur qui suit la lecture est immédiate pour les percussionnistes (Nathan, Diego, Fatou, Maria).
  • Le transport hérité est parfait dès l'arrivée — volume, sourdine 1-clic, continuité du tempo, trilingue FR/EN/PT : parité totale avec les autres pages, zéro réapprentissage.
  • Deux modes assumés : hors-ligne (lien / import, sans compte) ET salle en ligne — Yanis, Julien et Olivier saluent la réutilisation du client Supabase existant, sans login ni dépendance neuve.
  • Impression « ma fiche » de pupitre : Camille et Fatou repartent avec une partie imprimée par élève — l'atelier « en pupitre » qu'elles réclamaient depuis deux pages.

🗺 Plan d'action recommandé

  1. P1 · Une entrée « à froid » — C1, le seul critique : cliquer la porte « En équipe » sans config mène à un écran vide (import JSON / lien / Team Spirit) où les novices s'arrêtent. Ajouter « ▶ Essayer avec un exemple » (config de démo) + expliquer le parcours fabrique→joue. Impact fort, effort moyen.
  2. P1 · Franciser / expliquer le jargon d'entrée — C2 : « config d'équipe », « importer un JSON », « backing », « Team Spirit » bloquent les 66 ans et + et les IT 1–2. « backing » → « accompagnement », « fichier d'équipe », une phrase « une répartition = qui joue quoi ». Impact fort, effort faible.
  3. P2 · Présence + départ synchronisé en salle — C3 + C4 : afficher qui est connecté, un accusé de départ, un top horodaté pour partir vraiment ensemble (le décompte suppose une réception simultanée). Impact moyen, effort moyen.
  4. P2 · Accessibilité des chips et du décompte — C5 : mode et joueur sont des boutons à état visuel seul (classe .on) ; role=radio/aria-checked + aria-live sur l'overlay de décompte. Impact moyen, effort faible.
  5. P2 · Passerelle apprendre → pupitre — C6 : mettre une fiche de leçon « en pupitre » (demande de Camille, Aïcha, Rosa) — le v1.1 déjà noté. Impact moyen, effort moyen.
  6. P3 · Finitions — libellés de voix + instrument (C7), +1 cran de police sur légende/statuts (C8), QR / raccourci de partage du lien (C9), message clair quand le réseau manque en ligne (C10), export PNG du pupitre (C11), compaction mobile pendant le décompte (C12). Efforts faibles à moyens.
